# Riftbook Playbook Visual System

This visual system defines how The Real Vibe Coding Playbook looks, reads, and grows.

The goal is not decoration.

The goal is to make every lesson easier to scan, easier to remember, and easier to extend.

Riftbook is a field guide for builders working with AI coding agents. Its visual system should feel practical, honest, and structured.

It should help the reader understand:
- Where they are
- What kind of lesson they are reading
- What to copy
- What to avoid
- What to try next
- What came from real experience

## Visual Principles

1. Field guide, not course platform
2. Practical, not motivational
3. Honest, not polished fake-success content
4. Scannable, not dense
5. Modular, not manually redesigned every time
6. GitHub-native first
7. SVG assets should support the Markdown, not replace it

## System Parts

- Cover cards
- Section header cards
- Lesson hero cards
- Signal badges
- Callout blocks
- Learning path cards
- Case study blocks
- Progress maps

## Source of Truth

Any new playbook lesson should follow:

- `templates/lesson-template.md`
- `design-system/badge-system.md`
- `design-system/lesson-layout.md`
- `design-system/project-truth-layer.md`
- `design-system/icon-policy.md`

If a lesson needs a new visual pattern, add it to the design system before using it repeatedly.
