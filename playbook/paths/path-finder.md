## Use the path correctly

1. Choose based on the problem you have today, not the role you want later.
2. Open only the first recommended resource.
3. Apply its checklist, prompt, or template to a real project.
4. Continue to the second resource only after completing the first action.
5. Return here when your bottleneck changes.

A path is useful only when it changes the next action. Do not turn it into another reading backlog.
