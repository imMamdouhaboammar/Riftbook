**This path is for you if:** You think in product terms — user value, scope decisions, what to build vs. what to cut — and you want to use coding agents to move faster on the right things.

The challenge for product-minded builders is scope discipline. Agents can build a lot very quickly. The question is always: should we be building this at all?

## The Product-Minded Builder's Key Question

Before asking an agent to build anything: **does this belong in the first version?**

Write the answer in `PRODUCT.md` under "First version." Then reference it every time you consider adding scope. Agents will build whatever you ask them to build — and they will make it sound reasonable. Product judgment is yours to provide.
