**This path is for you if:** You run multiple projects or clients simultaneously — and you want to use coding agents in a way that is scalable, consistent, and not dependent on you remembering everything.

Agency operators have a different problem than solo builders: they cannot hold the full context of every project in their head at once. The system needs to hold it.

## The Agency Operator's Core Principle

**The system works. You oversee.**

Your job is to build a system that can handle project execution reliably — then spend your energy on the work that requires judgment that only you can provide: client communication, scope decisions, quality calls, and strategic direction.

If you are manually managing every agent interaction on every project, you have not built a system. You have built a complicated job.

**What makes this scalable:**
- Standardized context files (`PRODUCT.md`, `RULES.md`, `TASKS.md`) per client
- A default stack that applies to all projects unless explicitly changed
- Review gates that run automatically, not just when you remember
- Clear handoff points where you audit and approve before the next phase
