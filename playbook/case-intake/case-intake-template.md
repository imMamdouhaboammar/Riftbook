# Case Intake: Field Notes

*Use this template to write down the raw facts of a build struggle or success. Don't worry about clean formatting here — focus on raw narrative truth.*

---

### 1. What were you trying to build?
* Describe the feature, script, or configuration you were implementing.*

### 2. What problem triggered this?
* Was there a bug, a slow workflow, an API rate limit, or a tool conflict?*

### 3. What did you try first?
* What was your initial thought or approach before things went wrong?*

### 4. Which agent/tool did you use?
* Name the primary agent (e.g. Claude Code, Cursor, Codex) and the context setup.*

### 5. Where did the workflow break?
* Where did the agent loop, start deleting files, hallucinate imports, or fail to resolve a bug?*

### 6. What did you misunderstand at the time?
* What was your wrong assumption about the tool, the repo, or the agent's behavior?*

### 7. What fixed the issue?
* Concretely, what steps, prompts, stack shifts, or constraint modifications resolved it?*

### 8. What repeatable rule came out of it?
* If you had to write a single rule in `RULES.md` to prevent this from happening again, what would it say?*

* Choose from: `RED FLAG`, `DON'T BREAK`, `WHEN TO SWITCH`, `PRO TIP`, `QUICK WIN`, `MY MISTAKE`, `BIG ISSUE`, etc.*
