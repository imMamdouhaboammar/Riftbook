# Labs

Labs are practical exercises that turn the playbook into real build practice.

A path tells you what to learn. A lab tells you what to do.

## Available labs

| Lab | Best for | Link |
|---|---|---|
| First Real Project Lab | Fresh vibe coders building one real project from idea to handoff | [Open](./first-real-project-lab.md) |
| Debugging and Recovery Lab | Builders recovering a broken flow without random patching or weakened tests | [Open](./debugging-recovery-lab.md) |

## Choose the right lab

- Start with the [First Real Project Lab](./first-real-project-lab.md) when you need a complete build sequence.
- Use the [Debugging and Recovery Lab](./debugging-recovery-lab.md) when a real project is already broken or repeated fixes have failed.

## How labs work

Each lab includes:

- Objective
- Setup
- Step-by-step tasks
- Copy-ready prompts
- Checkpoints
- Final handoff

Use a real project and preserve evidence from each checkpoint. A lab is complete only when its final gate passes.
