# Review Guide

Use this guide when reviewing Riftbook changes.

## Fit

- The entry solves a real workflow problem.
- The category is correct.
- The entry is not a duplicate.
- The entry is more than a link.

## Source

- Official links are preferred.
- Links should be direct and clean.
- Claims should stay practical.

## Workflow

- The card explains when to use the resource.
- The card explains when not to use it.
- Setup or usage steps are clear.
- The reader knows what to do next.

## Agent value

- Agent instructions are included when useful.
- Expected output is clear.
- Review steps are included.

## Repo hygiene

- Category README is updated.
- Root README is updated for major resources.
- Placeholder sections are removed.
- Generated files are not included by accident.
