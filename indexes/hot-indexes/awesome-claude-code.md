## What it is

Awesome Claude Code is a large curated index of resources for Claude Code.

It collects Claude Code skills, agents, hooks, slash commands, status lines, orchestrators, applications, plugins, developer tooling, and related resources.

Official repo:
[https://github.com/hesreallyhim/awesome-claude-code](https://github.com/hesreallyhim/awesome-claude-code)

## Why it is a Hot Index

Awesome Claude Code is a Hot Index because it gives builders one of the fastest ways to discover the Claude Code ecosystem.

Instead of searching GitHub manually for skills, commands, hooks, agents, plugins, and workflow ideas, this repo acts as a discovery map. It is especially useful when you already use Claude Code and want to see what people are building around it.

This is not a single tool to install. It is a reference index for finding useful Claude Code extensions, patterns, and experiments.

## Best use cases

### 1. Discovering Claude Code resources

Use it when you want to find new skills, hooks, commands, agents, plugins, or workflow ideas for Claude Code.

### 2. Building your Claude Code setup

Useful when you are designing your own `.claude/` structure and want examples from the community.

### 3. Finding inspiration for agentic workflows

Good for seeing how other builders structure commands, agents, hooks, and tooling.

### 4. Comparing Claude Code ecosystem tools

Useful when you want to compare multiple resources before adding anything to your own setup.

### 5. Sourcing future cards for Awesome Vibe Coding

This can be used as a discovery source for new Tool Cards, Skill Cards, Hot Skills, Best Practice Guides, and Premium Picks.

## Important note

The upstream README is currently being reorganized.

Treat this repo as a discovery index, not as a polished course or final structured guide. When you find a useful item inside it, open the original resource, inspect its README, check maintenance signals, and decide whether it deserves a dedicated card in Awesome Vibe Coding.

## Quality checklist

Before adding any resource discovered from this index, check:

* The original repo has a clear README
* The use case is specific
* The install steps are understandable
* The resource is not abandoned
* The resource is not just hype or a thin wrapper
* The resource fits one of the existing Awesome Vibe Coding categories
* The card includes limitations and when not to use it
* The resource is tested or reviewed before being marked as Hot Skill or Premium Pick

## Notes

Awesome Claude Code is best used as a source of leads.

Do not copy its entire list into Awesome Vibe Coding. That would make your repo noisy. Use it to discover candidates, then create high-quality cards only for the resources that pass your own curation rules.
