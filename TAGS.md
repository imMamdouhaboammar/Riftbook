# Riftbook Tags

This file explains the main tags used across Riftbook cards.

Tags should help readers discover resources by workflow, not just by technology name.

## Core workflow tags

| Tag | Meaning |
|---|---|
| `ai-agents` | Resources for agent behavior, agent workflows, or agent infrastructure |
| `coding-agents` | Resources designed for Claude Code, Codex, Cursor, Gemini CLI, or similar coding agents |
| `workflow` | Step-by-step process for repeatable work |
| `review` | Code review, content review, or workflow review |
| `debugging` | Root-cause analysis and error resolution |
| `shipping` | Preparing work for production or delivery |

## Tool category tags

| Tag | Meaning |
|---|---|
| `mcp` | Model Context Protocol servers or MCP-related workflows |
| `cli` | Terminal-first tools |
| `github-action` | GitHub Actions and CI workflows |
| `browser-extension` | Browser-based tools or extensions |
| `local-first` | Tools that can run locally or keep important work local |

## Graph intelligence tags

| Tag | Meaning |
|---|---|
| `knowledge-graph` | Graph-based representation of files, facts, entities, or relationships |
| `graph-rag` | Graph-based retrieval and corpus reasoning |
| `project-intelligence` | Repo maps, architecture context, or codebase understanding |
| `review-intelligence` | Review context, blast radius, risk, affected flows, or test gaps |
| `agent-memory` | Long-running memory for agents |
| `temporal-memory` | Memory where facts change over time |
| `provenance` | Source-backed facts and traceable context |

## Frontend and design tags

| Tag | Meaning |
|---|---|
| `frontend-design` | UI, visual quality, layout, and design systems |
| `react-quality` | React-specific checks and quality gates |
| `ui-audit` | Reviewing existing UI for quality problems |
| `anti-slop` | Reducing generic or low-quality AI output |

## Context and token tags

| Tag | Meaning |
|---|---|
| `token-efficiency` | Reducing noisy context or verbose output |
| `context-compression` | Making context smaller and easier for agents to use |
| `context-layer` | Structured project or workflow context for agents |

## Research and learning tags

| Tag | Meaning |
|---|---|
| `research` | Research workflows, recent signal checks, or evidence gathering |
| `learning-path` | Structured learning resources |
| `guide` | Setup or best-practice documentation |
| `index` | Discovery hub or curated resource list |

## Tier tags

| Tag | Meaning |
|---|---|
| `premium-pick` | Strong default recommendation with repeated workflow value |
| `top-selected` | High-quality resource for a major use case |
| `core-framework` | Architecture-level system |
| `use-case-tool` | Narrow but useful tool |
| `hot-skill` | Skill that visibly improves agent output |
| `experimental` | Worth watching, but needs care or more testing |

## Tagging rules

- Use tags that describe workflow value.
- Avoid adding tags only for decoration.
- Prefer fewer, clearer tags.
- Add new tags here before using them widely.
- Do not use tags that duplicate the resource title without adding discovery value.
