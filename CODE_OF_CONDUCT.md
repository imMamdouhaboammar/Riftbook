# Code of Conduct

Riftbook is a practical repository for AI-assisted builders.

Contributions should stay useful, respectful, and focused on helping people build better workflows.

## Expected behavior

- Be clear and constructive.
- Keep feedback focused on the work.
- Explain why a change helps the repo.
- Prefer practical examples over broad opinions.
- Respect maintainers, contributors, and users.

## Unwelcome behavior

- Personal attacks
- Harassment
- Spam
- Off-topic promotion
- Repeated low-quality submissions
- Contributions that ignore the curation rules

## Review culture

Riftbook reviews are strict about usefulness, not status.

A contribution may be declined if it is unclear, too generic, duplicated, poorly sourced, or not useful in a real workflow.

That is not personal. It protects the quality of the repo.

## Reporting issues

If a discussion becomes unhelpful, open a maintainer note in the relevant issue or pull request and explain the concern clearly.

## Maintainer standard

Maintainers should apply the same curation and behavior standards to everyone.
